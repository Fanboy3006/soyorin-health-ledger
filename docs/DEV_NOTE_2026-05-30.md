## 二、新需求（测试阶段效率优化）

### 需求描述

在测试阶段，Web UI 输出的内容（今日汇总、流水列表、BMR 分析等）需要同步生成一个 Markdown 版本，方便 AI 审核，用户无需截图，直接粘贴 MD 文件内容即可给出评论和指导意见。

### 具体要求

- 每次页面刷新或每次录入/撤销操作后，自动生成（或更新）一个 MD 文件
- MD 文件内容包含：
  - 当前日期
  - 摄入/消耗/净热量汇总
  - BMR 估算 + 热量差
  - 今日流水列表（饮食 + 训练）
  - 当前所有激活的预设列表（可选）
- 文件保存位置：本地项目目录下的 `test_logs/YYYY-MM-DD_HH-MM-SS.md` 或固定文件名如 `latest_test_state.md`
- 用户可一键复制 MD 内容到剪贴板

### 价值

- 省去截图 + 手打描述的时间
- AI 可直接读取结构化 MD 内容，精准理解当前 UI 状态
- 便于版本回溯和对比
