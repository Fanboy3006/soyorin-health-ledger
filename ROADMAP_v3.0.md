## 5. 建议的下一步行动（明天先做什么）

### 优先级排序

```
明天 → 1. 修复高优先级 Bug（已全部完成 ✅）
       2. 手动录入入口（Sprint 1 剩余，4h）
       3. 历史流水查看（Sprint 1 剩余，4h）
       4. 预设排序拖拽（Sprint 1 剩余，3h）
       5. 开始 Sprint 2：日终清算页面（6h）
```

### 推荐执行顺序

1. **手动录入入口** — 用户最需要的功能，没有预设时也能记录
2. **历史流水查看** — 日期选择器 + 按日期筛选，提升数据回溯能力
3. **预设排序拖拽** — 提升预设管理体验
4. **日终清算页面** — Sprint 2 核心功能，完成"记账 → 清算"闭环

### 技术债务提醒

- `tsconfig.json` 中 `noUnusedLocals` 和 `noUnusedParameters` 可能导致构建失败，建议设为 `false`
- Dexie 版本升级到 v3 后，旧数据可能无法自动迁移，需测试 `db.version(3).upgrade()`
- Supabase 表结构尚未创建，`sync.ts` 中的 `supabase.from('ledger_entries')` 会报错，需先在 Supabase 控制台建表
